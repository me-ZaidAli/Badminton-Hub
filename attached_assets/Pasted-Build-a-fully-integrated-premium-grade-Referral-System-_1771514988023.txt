Build a fully integrated, premium-grade Referral System for the Badminton Performance Group app using unique, single-use referral codes and shareable referral links. Each user must have a dedicated Referrals section accessible from the main menu, user profile, and optionally highlighted on the main dashboard as a polished “Refer & Earn” promotional card, designed with modern cards, progress bars, status chips, and a professional, impressive UI. From this section, users can generate a new referral code and referral link for each referral attempt; each code must be system-generated, single-use only, expire automatically after 30 days, and display a visible countdown and status (Active, Used, Expired). The referral link must redirect to the login/signup page, where the signup form includes an optional referral code field; the system must validate that the referral code exists, is unused, not expired, and matches the referred person’s name and email before allowing referral-based registration. Once completed, the referral is logged as Pending Approval, visible in the referrer’s referral dashboard and the Admin Referral Management tab. Upon referral submission, automatically send the referrer an internal chat message thanking them for the referral and an in-app notification confirming the referral is pending. Admins must be able to approve or reject referrals from a dedicated Admin Referrals tab; approving a referral automatically marks the referral code as used, adds £4 credit to the referrer’s credit wallet, logs the action, updates referral progress, and triggers both an internal chat message and in-app notification confirming approval and credit awarded. Credits must accumulate automatically, with the system tracking reward thresholds and notifying users when milestones are reached: £8 credit triggers eligibility for a 4-month Premium member rate, £16 reflects 4 approved referrals, and eligibility must be clearly flagged for Admins to apply benefits manually. The user referral dashboard must show total referrals, referral history, credit balance, reward progress (e.g. 2/4 referrals), unlocked benefits, expired referral alerts, and a timeline of referral activity. The system must notify users when referral codes expire unused and allow generation of new codes at any time. Admins must have full visibility to see how many referrals each user has made, credits issued, reward eligibility, and referral statuses, with filters, audit logs, permission enforcement, and duplicate-referral prevention via email matching. Ensure the entire referral experience feels premium, transparent, motivating, and seamless for both users and admins.
